정적 단일 페이지 웹앱 “캐릭터 프롬프트 빌더”를 만들어줘.
기술 스택은 HTML/CSS/Vanilla JS만 사용하고, 프레임워크/외부 라이브러리/백엔드는 사용하지 마.
한 파일(index.html)로 실행 가능한 형태로 작성해.

[핵심 목표]
- 사용자가 30개의 ‘친절한 질문’에 대한 보기(드롭다운)를 선택하면
  ① GPT 프롬프트, ② BananaAI/일반 LLM 프롬프트, ③ Midjourney/SDXL 프롬프트
  를 자동 생성하고 복사할 수 있게 한다.
- 모든 UI는 한 페이지(스크롤)에서 동작. 탭 전환 없음.
- 한국어 라벨/설명을 사용.

[화면 구성]
1) 상단 헤더: 제목, 간단 설명.
2) 질문 섹션: 아래 30개 ‘친절한 질문’을 모두 드롭다운으로 나열한다.
   - 드롭다운 표시는 “A. 텍스트” 형식, 저장값은 코드(A/B/…).
   - 미선택 문항은 프롬프트 요약에서 제외(부분 응답 허용).
3) 옵션 입력: 스타일 힌트, 네거티브 힌트, 이미지 비율/버전(예: --ar 3:4 --v 6).
4) 버튼: “프롬프트 생성”, “전체 복사”, “선택 저장”, “불러오기”, “JSON 내보내기”, “JSON 가져오기”.
5) 결과 영역: GPT/LLM/Midjourney용 각각 textarea + “복사” 버튼.
6) 저장/불러오기: localStorage 사용.
7) JSON 내보내기 형식:
   { "character_profile": { "1": {"answer":"C"}, "2": {"answer":"E"}, ... } }

[프롬프트 생성 규칙]
- 선택된 코드(A/B/…)를 사람이 읽는 텍스트로 매핑하여 “설정 요약”을 만든다.
- 미선택 문항은 요약에서 제외.
- 생성 결과는 아래 3개를 각각 textarea에 출력하고, 각 textarea에 “복사” 버튼을 둔다.

① GPT 프롬프트(한국어):
“당신은 캐릭터 디자이너입니다. 아래 설정을 바탕으로 한국어 캐릭터 시트를 작성하세요.
[설정 요약]
섹션: 1) 개요(한 문장) 2) 세계관 3) 외형(키/체형/색감/질감/제스처) 4) 성격(가치관/말투/버릇)
5) 능력과 한계 6) 관계·감정선 7) 목표·갈등 8) 상징·테마 9) 핵심 대사 3줄 10) 이미지 프롬프트(1줄).
문체: 간결하지만 생생하게, 클리셰 최소화. 누락 정보는 상식선에서 창의적으로 보완.”

② BananaAI/일반 LLM 프롬프트(한국어):
“SYSTEM: 너는 세계관/캐릭터 기획 전문가다.
USER: 아래 설정을 바탕으로 600~900자 캐릭터 설명을 한국어로 작성해줘.
포함: 세계관, 외형, 성격, 목표, 약점, 테마, 캐치프레이즈.
[설정 요약]”

③ Midjourney/SDXL 프롬프트(간결):
- form:[11], look&texture:[12], size:[13], symbol:[15], color:[26], season:[27], mood:[6], theme:[29]
- 스타일 힌트(있으면 콤마로 추가)
- 네거티브 힌트는 문장 끝에 “--no …”
- 이미지 비율/버전 옵션은 문장 끝에 그대로 추가(예: “--ar 3:4 --v 6”)
- 전체 1~2문장 내의 간결한 문장

[버튼 동작]
- 프롬프트 생성: 위 규칙으로 3개 결과 생성.
- 전체 복사: 3개 결과를 ‘---’로 구분해 한 번에 클립보드 복사.
- 선택 저장/불러오기: localStorage 사용(질문 답변 + 옵션 3종 저장).
- JSON 내보내기/가져오기: 위 명세 형식으로 저장/복원.

[디자인]
- 다크 테마, 가독성 높은 카드형 레이아웃, 모바일 반응형.
- 포커스 하이라이트 제공. 한국어 라벨.
- 하단 푸터: “© 2025 Character Prompt Builder – Replit Static App”

[동작 조건]
- Run 클릭 시 즉시 실행(정적 페이지). 외부 의존성 및 백엔드 금지.
- 결과물은 index.html 한 파일로 제출.
